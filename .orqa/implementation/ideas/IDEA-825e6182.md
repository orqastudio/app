## What's Missing

The SDK (`libs/sdk/`) is the primary API surface for plugins and the app UI. It exposes:

### Stores (12)

- `NavigationStore` тАФ routing, activity panel, artifact selection, hash-based navigation
- `ArtifactStore` тАФ nav tree, artifact loading, CRUD
- `SessionStore` тАФ conversation sessions, messages, streaming
- `ProjectStore` тАФ project settings, project path
- `ConversationStore` тАФ active conversation state
- `SettingsStore` тАФ user preferences, active sections
- `EnforcementStore` тАФ validation, integrity checks
- `ErrorsStore` тАФ error collection, display
- `LessonsStore` тАФ lesson pipeline
- `SetupStore` тАФ setup wizard state
- `ToastStore` тАФ notifications
- `PluginRegistry` тАФ plugin discovery, view registration

### Router

- `pushRoute` / `replaceRoute` тАФ hash-based routing
- `parseHash` / `currentRoute` тАФ route parsing
- `injectNavigation` тАФ SvelteKit integration (injection pattern)
- `navigateToArtifact` / `navigateToPluginView` / `navigateToActivity` etc.

### IPC

- Tauri invoke wrappers
- Type-safe command interfaces

### Plugin System

- View registration
- Shared module exposure (`window.__orqa`)
- Plugin lifecycle

### Utilities

- Frontmatter parsing
- Logger

## What's Needed

1. **SDK Reference Guide** тАФ each store's API, properties, methods, usage examples
2. **Plugin Developer Guide** тАФ how to use SDK from a plugin (navigation, data access, views)
3. **Knowledge artifacts** тАФ for agent context when building plugins or app UI
4. **Architecture doc** тАФ the injection pattern, store initialization lifecycle, data flow
