## What

`libs/validation/` is a misleading name for what is actually the core runtime engine:

- Artifact graph builder and query engine
- Validation and integrity checking
- Search engine (ONNX + DuckDB)
- HTTP daemon serving all of the above
- Backend for MCP server, LSP server, CLI, and app

It should be a top-level directory (e.g., `engine/` or `daemon/`) with naming that makes its critical-path role obvious.

## Why

- New contributors look at `libs/validation/` and think "validation helper library"
- The daemon binary name (`orqa-validation`) doesn't communicate "this is the core engine"
- Finding daemon code requires knowing it's buried in a library subdirectory
- The component's structural importance should match its runtime importance
