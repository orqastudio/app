## Prerequisites

MS-b1ac0a20 (Dogfooding) is a waypoint on this path, not a separate destination. Its remaining work feeds directly into this milestone.

## Target User

Non-technical thinkers: product managers, consultants, researchers, solo founders, team leads — anyone who needs to turn messy situations into structured understanding and evolving plans. They don't write code. They don't use terminals. They experience OrqaStudio through the desktop app, the conversation panel, and the artifacts it produces.

The MVP must also support dogfooding OrqaStudio to build OrqaStudio. DevTools are included for this purpose.

## Three Irreducible Capabilities

Remove any one and it's no longer OrqaStudio.

### 1. The Artifact Graph (SurrealDB)

Structured, typed, related artifacts forming a navigable knowledge graph. SurrealDB as source of truth — not markdown files scanned into a HashMap. 32 relationship types, forward-only with computed inverses, graph traversals, health metrics.

### 2. The Learning Loop

Lesson capture → recurrence detection → rule promotion → prompt injection → behaviour change. The system genuinely gets smarter over time. The user can see it learning.

### 3. The Methodology Layer

Opinionated structured thinking shipped as composable methodology via the plugin framework (P1). Entry modes guide new users in. Seven thinking modes route work. Everything is a plugin.

## What Is Explicitly Post-MVP

The MVP architecture must not preclude these, but they are not part of the foundation.

- **Connectors** — Claude Code plugin, Cursor rules, Copilot instructions
- **Multi-provider AI** — providers beyond Claude (architecture is provider-agnostic)
- **Scale infrastructure** — self-hosted/cloud tiers, Postgres, multi-user collaboration
- **Plugin ecosystem** — community plugins, marketplace, three-way diff
- **Advanced enforcement** — pre-commit gates, behavioural rule enforcement beyond prompt injection

## Completion Criteria

- [ ] A non-technical user can install the app and get value within 10 minutes
- [ ] Conversations produce structured artifacts that persist and relate to each other
- [ ] The artifact graph is navigable — users can see how their thinking connects
- [ ] The learning loop visibly closes — the user can see the system getting smarter
- [ ] The methodology guides thinking without requiring agile terminology
- [ ] The plugin architecture works end-to-end (core methodology plugin ships)
- [ ] DevTools provide enough visibility for effective dogfooding
- [ ] App is distributable (signed installers for Windows, macOS, Linux)
- [ ] The architecture is ready to grow into connectors, multi-provider, multi-user, and ecosystem without rewrites
