# TASK-bc924b75: Run orqa install and validate generated agents

## What to do

Run `orqa install` end-to-end and verify the agent file generation pipeline:

1. **Run orqa install:**
   - Execute `orqa install` from the project root
   - Capture output — look for any errors or warnings
   - Verify exit code 0

2. **Validate generated agents:**
   - List all files in `.claude/agents/*.md`
   - For each file, verify it was generated by the pipeline:
     - Check for a generation marker or consistent structure matching pipeline output
     - Compare against `.orqa/prompt-registry.json` — every agent in the registry should have a corresponding file
     - Every file should match registry content, not contain hand-written overrides

3. **Check for hand-maintained files:**
   - Identify any `.claude/agents/*.md` files NOT in the prompt registry
   - These are hand-maintained and should be migrated to the pipeline
   - Flag each with the content that should be in the registry instead

4. **Verify pipeline data flow:**
   - `.orqa/prompt-registry.json` is populated by install
   - Agent files read from the registry
   - No bypass path exists

5. **Run build checks:**
   - `npx tsc --noEmit` for `libs/cli`
   - `cargo clippy -- -D warnings`

## Knowledge needed

- `libs/cli/src/commands/install.ts` — install command implementation
- `libs/cli/src/lib/agent-file-generator.ts` — agent file generator
- `.orqa/prompt-registry.json` — prompt registry data
- `.claude/agents/*.md` — generated output files

## Agent role

Researcher — runs install command, reads output files, produces findings. No source code changes.

## Verification

- `orqa install` exit code 0
- Count of agent files matches count of agents in prompt-registry.json
- Diff between generated and existing files shows pipeline consistency
- `npx tsc --noEmit` passes
- `cargo clippy -- -D warnings` passes
