# TASK-5a858776: Design service dependency checks

## What to do

Design the connector's startup checks for all three required services:

1. **Daemon-gate (verify existing):**
   - Locate the daemon-gate in `connectors/claude-code/src/`
   - Verify it blocks session start when daemon is not running
   - Document the mechanism (HTTP health check, PID file, port check)

2. **MCP server check (design new):**
   - Design a check mechanism for MCP server availability
   - Options: HTTP/WebSocket health endpoint, port connectivity test
   - During migration: check should be designed but may validate against code quality rather than running services
   - Document: endpoint, timeout, retry strategy, failure behavior

3. **LSP server check (design new):**
   - Design a check mechanism for LSP server availability
   - Options: stdio initialization handshake, port connectivity test
   - During migration: same as MCP — design-phase, not fully operational yet
   - Document: endpoint, timeout, retry strategy, failure behavior

4. **`orqa dev` command design:**
   - Design the unified dev command that spawns all three services
   - Daemon: spawns with file watchers for rule/workflow changes
   - MCP: spawns with connection to daemon
   - LSP: spawns with connection to daemon
   - Watch mode: restart on relevant file changes
   - Document: spawn order, health check sequence, graceful shutdown

5. Store the design as an AD (architecture decision) or research artifact in `.orqa/`
