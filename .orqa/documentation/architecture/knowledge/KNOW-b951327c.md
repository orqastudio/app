# Agent Architecture and Prompt Generation Pipeline

## Base Roles (8 Fixed)

| Role | Responsibility | Artifact Scope |
| ------ | --------------- | ---------------- |
| **Orchestrator** | Coordinates, delegates, reads summaries | Read-only |
| **Implementer** | Writes code, runs tests | Source code, shell access |
| **Reviewer** | Verifies quality, produces PASS/FAIL verdicts | Read-only, checks only |
| **Researcher** | Investigates, gathers information | Read-only, creates research artifacts |
| **Writer** | Creates/edits documentation | Documentation only |
| **Planner** | Designs approaches, maps dependencies | Plans and delivery artifacts |
| **Designer** | Creates UI/UX, component structures | Design artifacts, component code |
| **Governance Steward** | Maintains governance artifacts | `.orqa/` artifacts only |

## Task-Specific Agent Generation

Agents are **generated on a bespoke basis** for each task — not pre-defined:

```text
Base Role + Workflow Context + Domain Knowledge = Task-Specific Agent
```text

Agents are **ephemeral** — generated and discarded. No agent artifact type, no AGENT-*.md files.

## Prompt Generation Pipeline (5 Stages)

```text
Plugin Registry → Schema Assembly → Section Resolution → Token Budgeting → Prompt Output
```text

| Stage | What Happens |
| ------- | ------------- |
| Plugin Registry | Installed plugins register prompt contributions at install time |
| Schema Assembly | Collect applicable sections for (base role, workflow, task) tuple |
| Section Resolution | Resolve references to compressed summaries; follow cross-refs depth 1 |
| Token Budgeting | Trim P3 first, then P2, then P1; never trim P0 |
| Prompt Output | Static core at TOP (cached), dynamic content at BOTTOM (changes per turn) |

**Accuracy over speed** — on-demand MCP retrieval latency is acceptable if it produces better outcomes.

## Injection Tiers

| Tier | When Delivered |
| ------ | --------------- |
| `always` | Compressed summary at every spawn |
| `stage-triggered` | When active workflow stage matches |
| `on-demand` | Via MCP semantic search when needed |
