Tauri v2 cross-platform app development: Rust backend commands, IPC patterns, capabilities, and common issue prevention.

## Quick Start

```rust
// 1. Create command in lib.rs
#[tauri::command]
fn greet(name: String) -> String { format!("Hello, {}!", name) }

// 2. Register in builder (commands NOT registered silently fail)
tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![greet])
    .run(tauri::generate_context!()).expect("error");
```text

```typescript
// 3. Call from frontend (use @tauri-apps/api/core, NOT /tauri)
import { invoke } from '@tauri-apps/api/core';
const result = await invoke<string>('greet', { name: 'World' });
```text

```json
// 4. Add capabilities (v2 denies everything by default)
{ "identifier": "default", "windows": ["main"], "permissions": ["core:default"] }
```text

## Critical Rules

- Register EVERY command in `generate_handler![]`
- Return `Result<T, E>` for error handling
- Use `Mutex<T>` for shared state
- Use owned types (`String` not `&str`) in async commands
- Use `lib.rs` for shared code (mobile builds require it)
- Never block the main thread — use async
- Never hardcode paths — use `app.path()`

## Known Issues

| Issue | Root Cause | Fix |
| ------- | ----------- | ----- |
| "Command not found" | Missing from `generate_handler!` | Add to macro |
| "Permission denied" | Missing capability | Add to capabilities |
| State panic | Type mismatch in `State<T>` | Match `.manage()` type |
| White screen | Frontend not building | Check `devUrl`/`beforeDevCommand` |
| IPC timeout | Blocking async | Use spawn/async |

## Key Config (tauri.conf.json)

- `build.devUrl`: Must match frontend dev server port
- `build.beforeDevCommand`: Frontend dev server start command
- `app.security.capabilities`: Array of capability identifiers
- `[lib] crate-type = ["staticlib", "cdylib", "rlib"]` in Cargo.toml for cross-platform

## Error Handling Pattern

```rust
#[derive(Debug, thiserror::Error)]
enum AppError {
    #[error("IO: {0}")] Io(#[from] std::io::Error),
    #[error("Not found: {0}")] NotFound(String),
}
impl serde::Serialize for AppError {
    fn serialize<S>(&self, s: S) -> Result<S::Ok, S::Error>
    where S: serde::ser::Serializer { s.serialize_str(&self.to_string()) }
}
```text
