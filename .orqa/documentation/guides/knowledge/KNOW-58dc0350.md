# Versioning System — Agent Reference

## Core Rule

Single canonical version for the entire ecosystem. All repos share one version number.

## VERSION File

Located at dev repo root. Single source of truth.

## Version Format

- **Development:** `X.Y.Z-dev` — rolling from main branch
- **Stable release:** `X.Y.Z` — tagged, no suffix
- **Pre-release:** `X.Y.Z-rc.N` — release candidates

The `-dev` suffix is mandatory for all pre-release work.

## Commands

```bash
orqa version sync          # sync VERSION to all repos
orqa version bump 0.2.0-dev  # bump and sync in one step
orqa version check         # flag repos where version doesn't match
```

Sync updates `version` in every `package.json`, `Cargo.toml`, `orqa-plugin.json`, and `plugin.json` across all submodules.

## Release Process

1. `orqa version bump X.Y.Z` (drop `-dev`)
2. `git tag vX.Y.Z`
3. `git push --tags`
4. Immediately bump to next dev: `orqa version bump X.Y.(Z+1)-dev`
