# Svelte Plugin Setup — Agent Reference

## Installation

```bash
orqa plugin install @orqastudio/plugin-svelte
```

## Installer Actions

1. Detects sub-projects using TypeScript/Svelte
2. Recommends which sub-projects should receive the plugin
3. Adds dev dependencies to each selected project's `package.json`
4. Runs `npm install`
5. Generates initial config files from coding standards rules

## Dependencies Added

- `eslint`, `@typescript-eslint/eslint-plugin`, `@typescript-eslint/parser`
- `eslint-plugin-svelte`
- `svelte-check`
- `vitest`, `@testing-library/svelte`
- `typescript`

Only missing dependencies are added — existing ones preserved.

## Organisation Mode

At org level, plugin asks which sub-projects apply. AI recommends based on detected languages. Each selected project gets dependencies installed and config generated from org-level rules. Sub-projects can override specific standards with tracked rationale.
