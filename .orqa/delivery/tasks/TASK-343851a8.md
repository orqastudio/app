## What

Recurring lessons and under-enforced rules represent systemic process failures. When a lesson is observed 3 or more times without being promoted to enforcement, it's a signal that the enforcement gap is real and persistent. This command automates detection of these gaps and creates escalation tasks to ensure they are addressed.

## Why

Per RULE-009 (dogfood mode enforcement priority), enforcement gaps are always CRITICAL. Manual auditing is not reliable — the tool must catch these automatically. Running as a stop hook ensures no session ends without surfacing new escalation candidates.

## How

### CLI command
1. Add `audit escalation` subcommand to the `orqa` CLI (Rust implementation in `libs/cli/`)
2. Scan logic:
   - Lessons: read all `LESS-*.md` files, filter where `recurrence >= 3` AND no `promoted-to` relationship pointing to a rule
   - Rules: read all `RULE-*.md` files, filter where `enforcement_gap_count >= 3` (or similar field) AND enforcement array is empty or `manual`-only
3. For each candidate, check if an escalation task already exists (idempotency)
4. Create `TASK-*.md` artifacts with `type: task`, `status: captured`, and a clear description of what needs enforcement

### Stop hook
1. Add entry to connector's `hooks.json` under `stop` or `session-end` event
2. Hook invokes `orqa audit escalation`
3. Any created tasks are reported to the user before session ends

## Verification

1. `orqa audit escalation --help` shows command documentation

4. The stop hook fires at session end and outputs the escalation report
