# Code Actions for Quick Fixes

## What to Implement

The LSP currently does not register a code action provider. This task adds one that offers quick fixes associated with diagnostics.

### Steps

1. **Register code action provider** in `initialize` response — set `code_action_provider` with supported code action kinds (`quickfix`).

2. **Implement `textDocument/codeAction` handler** — for each diagnostic in the requested range, generate applicable code actions:

   - **Invalid status value** — offer to replace with the nearest valid status from plugin schemas
   - **Missing required field** — offer to insert the field with a default/placeholder value
   - **Broken artifact link** — offer to remove the broken reference, or suggest the closest matching valid ID
   - **Missing inverse relationship** — offer to add the inverse to the target file (shared with TASK-aa9aec9e)

3. **Generate `WorkspaceEdit` for each action** — code actions return edits that the editor can apply in one click.

4. **Support "fix all" actions** — when multiple diagnostics of the same type exist, offer a single action to fix all of them.

## Acceptance Criteria

- [ ] Code action provider is registered in LSP capabilities
- [ ] Invalid status diagnostics offer quick fix to replace with valid status
- [ ] Missing required field diagnostics offer quick fix to insert the field
- [ ] Broken link diagnostics offer quick fix to remove or replace the reference
- [ ] Code actions generate correct `WorkspaceEdit` that can be applied by the editor
- [ ] "Fix all" actions are available when multiple diagnostics share a type
- [ ] No `unwrap()` / `expect()` / `panic!()` in new code
- [ ] `make lint-backend` passes with zero warnings
