## Context

Session 2026-03-25 discovered that:

1. **No plugin declares any relationship types** — every `provides.relationships` is `[]`
2. **The schema infrastructure exists** — `PluginRelationshipExtension` in `libs/types/src/platform/plugin-extension.schema.json` is fully designed with `key`, `inverse`, `from`, `to`, `semantic`, `constraints`
3. **The validator skips when vocabulary is empty** — `build_valid_relationship_types()` returns an empty set, so all types pass
4. **Unapproved types proliferated** — `related`, `grounded`, `cautions`, `crystallised-by`, `teaches`, `synchronised-with`, `complements` were never approved by user
5. **`project.json` declares `depends-on`** — wrong location, should be in delivery plugin

## Design Note

Plugins extend an empty base schema. The schema SHAPE is defined locally (in libs/types), but the actual content (which relationship types exist) lives with the plugins that own those artifacts. The core plugin doesn't define relationship types — only the engine that evaluates them.
