## What

Verify Principle 4 from RES-d6e8ab11 section 2:

> **P4: Declarative Over Imperative** — State machines, guards, actions, and workflow definitions are YAML declarations validated by JSON Schema -- not code. Plugin authors write configuration, not functions. This keeps state machines portable, diffable, and inspectable. Guard logic uses declarative primitives (field checks, relationship checks, graph queries), not arbitrary functions.

Confirm that all workflow definitions use declarative YAML, not imperative code.

## Knowledge Needed

- `.orqa/discovery/research/RES-d6e8ab11.md` section 2 (P4 definition)
- `plugins/*/workflows/*.yaml` — all plugin workflow files
- `.orqa/workflows/*.resolved.yaml` — resolved workflow files
- `libs/cli/src/` — workflow engine code (should process YAML, not contain definitions)
- `libs/daemon/src/` — daemon workflow evaluation (should interpret declarative guards)

## Agent Role

Researcher — read-only audit producing a PASS/FAIL verdict with evidence.

## Steps

1. Read RES-d6e8ab11 section 2 to confirm the exact P4 principle text
2. List all workflow YAML files across all plugins: `plugins/*/workflows/*.yaml`
3. List all resolved workflows: `.orqa/workflows/*.resolved.yaml`
4. Review each workflow file for declarative structure: states, transitions, guards as field checks or relationship checks
5. Search for any `code_hook` entries in workflow files — document and assess whether they are justified
6. Search for any imperative logic (functions, callbacks, arbitrary code) embedded in workflow definitions
7. Verify that guard logic uses declarative primitives: `field-check`, `relationship-check`, `graph-query` — not function references
8. Check the daemon's workflow evaluation code to confirm it interprets declarative guards, not executes arbitrary code
9. Document all workflow files with their structure (declarative vs imperative)
10. Produce a PASS/FAIL verdict

## Verification

- `find plugins/ -name "*.workflow.yaml" | wc -l` — total workflow file count
- `grep -rn "code_hook\|function\|callback\|execute" plugins/ --include="*.yaml"` — should be 0 or documented exceptions
- All guard entries use declarative primitives (field-check, relationship-check, graph-query)
- JSON Schema validation exists for workflow YAML structure
