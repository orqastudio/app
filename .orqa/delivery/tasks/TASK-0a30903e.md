## What

Update `plugins/agile-planning/` (created in TASK-3) to define planning-stage artifact types. These types have different lifecycles and semantics from their discovery-stage counterparts. A `planning-idea` is a scoped implementation approach, a `planning-research` is technical feasibility analysis, and a `planning-decision` is a concrete architectural or implementation decision.

Also move `wireframe` from software-kanban into agile-planning — wireframes are planning artifacts that refine discovery decisions into concrete designs.

## Knowledge Needed

- Read `plugins/agile-planning/orqa-plugin.json` (created in TASK-3) for current manifest
- Read `plugins/agile-discovery/orqa-plugin.json` (created in TASK-5) for cross-stage reference targets
- Read `plugins/software-kanban/schemas/` for wireframe schema to move
- Read `plugins/software-kanban/workflows/wireframe.workflow.yaml` for wireframe workflow to move
- Read existing schema files for format reference (e.g., `plugins/agile-discovery/schemas/discovery-idea.json`)
- Read existing workflow files for state machine format reference

## Agent Role

Implementer — add schemas, create workflows, move wireframe, define relationships.

## Steps

1. Create planning-stage schemas in `plugins/agile-planning/schemas/`:
   - `planning-idea.json` — scoped implementation approach
   - `planning-research.json` — technical feasibility analysis
   - `planning-decision.json` — concrete architectural/implementation decision
2. Move wireframe schema from `plugins/software-kanban/schemas/` to `plugins/agile-planning/schemas/`
3. Create standalone workflows in `plugins/agile-planning/workflows/`:
   - `planning-idea.workflow.yaml` with planning-appropriate states (draft -> evaluating -> accepted/rejected)
   - `planning-research.workflow.yaml` with states (proposed -> investigating -> concluded)
   - `planning-decision.workflow.yaml` with states (proposed -> reviewing -> resolved/deferred)
   - Move `wireframe.workflow.yaml` from software-kanban
4. Update `plugins/agile-planning/orqa-plugin.json`:
   - Add all 4 types to `provides.schemas`
   - Add all 4 workflows to `provides.workflows`
   - Define relationship types with cross-stage joins:
     - planning-idea -> planning-research (evolves-to)
     - planning-research -> planning-decision (evolves-to)
     - planning-decision -> discovery-research (references, cross-stage)
     - planning-decision -> epic (implements, cross-stage to implementation)
     - wireframe -> planning-decision (visualizes)
5. Remove wireframe from `plugins/software-kanban/` manifest and files
6. Run `npx tsc --noEmit` in `libs/cli`
