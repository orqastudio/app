## What

Three independent schema validation implementations exist:
1. **Rust** (`libs/validation/src/checks/schema.rs`) — canonical, uses `jsonschema` crate, auto-derives `id` patterns and `status` enums from type definitions
2. **App hooks** (`app/.githooks/validate-schema.mjs`) — uses `ajv` v8 + `ajv-formats`, loads `schema.json` files directly
3. **Plugin hooks** (`plugins/githooks/hooks/validate-frontmatter.mjs`) — uses `ajv`, has its own `buildJsonSchema()` that mirrors the Rust version

The Rust version auto-derives constraints the JS versions do not know about. Validation results can diverge.

## How

1. Migrate `app/.githooks/validate-schema.mjs` to call daemon `POST /parse` with each staged file path
2. Migrate `plugins/githooks/hooks/validate-frontmatter.mjs` similarly
3. Follow the `validate-artifact.ts` pattern: read stdin JSON for hook context, call daemon, return findings
4. Handle daemon-unavailable gracefully (warn but allow, or fall back to binary)
5. Remove ajv dependencies from hook scripts if no other consumers need them

## Files

- `app/.githooks/validate-schema.mjs` — app hook to migrate
- `plugins/githooks/hooks/validate-frontmatter.mjs` — plugin hook to migrate

- `connectors/claude-code/src/hooks/validate-artifact.ts` — reference pattern
