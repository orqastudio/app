## What

Verify Principle 1 from RES-d6e8ab11 section 2:

> **P1: Plugin-Composed Everything** — No governance pattern is hardcoded in the core framework. Workflow stages, artifact types, state machines, knowledge artifacts, and agent specializations all come from plugins. The core framework provides engines, not definitions. This makes OrqaStudio adaptable to any domain without forking.

Audit the core framework code in `libs/` to confirm that no governance definitions are hardcoded. The core should provide engines (graph engine, workflow engine, state machine engine, prompt pipeline) but NOT definitions (artifact type names, status values, relationship type strings, knowledge content).

## Knowledge Needed

- `.orqa/discovery/research/RES-d6e8ab11.md` section 2 (P1 definition)
- `libs/cli/src/` — the core framework code to audit
- `libs/daemon/src/` — daemon code to audit
- Plugin manifests in `plugins/*/orqa-plugin.json` — what definitions plugins provide

## Agent Role

Researcher — read-only audit producing a PASS/FAIL verdict with evidence.

## Steps

1. Read RES-d6e8ab11 section 2 to confirm the exact P1 principle text
2. Search `libs/cli/src/` for hardcoded artifact type names (e.g., literal strings like "epic", "task", "knowledge", "rule" used as type identifiers rather than read from schema/config)
3. Search `libs/cli/src/` for hardcoded status values (e.g., "active", "done", "blocked" as literal strings rather than schema-derived)
4. Search `libs/cli/src/` for hardcoded relationship type strings (e.g., "delivers", "depends-on" as literals rather than schema-derived)
5. Search `libs/daemon/src/` for the same patterns
6. Distinguish between engine code (which processes types generically) and definition code (which names specific types) — only the latter is a violation
7. Document every finding with file path, line number, and whether it is a violation or false positive
8. Produce a PASS/FAIL verdict

## Verification

- `grep -rn '"epic"\|"task"\|"knowledge"\|"rule"\|"lesson"' libs/cli/src/ --include="*.ts" | grep -v "// \|test\|spec\|\.d\.ts"` — review each match for hardcoded type usage
- `grep -rn '"active"\|"done"\|"blocked"\|"draft"' libs/cli/src/ --include="*.ts" | grep -v "// \|test\|spec\|\.d\.ts"` — review each match for hardcoded status usage
- Same searches in `libs/daemon/src/` with `--include="*.rs"`
- Count of violations must be 0 for PASS
