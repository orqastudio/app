## Scope

### Process to validate

For each repo:

1. Clone the repo locally (bare or full)
2. Run `git filter-repo --to-subdirectory-filter <target-path>` — moves all files into the target subdirectory, rewriting history
3. In the target monorepo: `git remote add <name> <local-clone>`
4. `git fetch <name>`
5. `git merge <name>/main --allow-unrelated-histories`
6. `git remote remove <name>`

### Test with subset first

Pick 3 diverse repos:

- `libs/types` (leaf TS lib, no deps)
- `connectors/claude-code` (connector with hooks and setup code)
- `plugins/software` (content plugin with views)

Run the process on a throwaway copy of the dev repo. Verify history, file placement, and no conflicts.

### Output

A shell script `scripts/monorepo-merge.sh` that automates the process for all 30 repos.
