# TASK-8809270b: Remove business logic from connector hooks

## What to do

Using the audit report from TASK-ff537cb6, refactor each business-logic hook to a thin adapter:

1. For each hook classified as "business logic" in the audit:
   - Remove the enforcement logic (if/else checks, field validation, governance decisions)
   - Replace with the thin adapter pattern: parse HookContext from stdin -> POST to daemon `/hook` endpoint -> format daemon response as output
   - Ensure the daemon has equivalent rule entries for the removed logic
2. Delete standalone enforcement scripts that duplicate daemon logic:
   - Check for files like `enforce-background-agents.mjs`, `enforce-completion-gate.mjs`
   - If the enforcement is already in daemon rules, delete the script
   - If not in daemon, add the rule entry type first, then delete the script
3. Verify every hook follows the pattern:
   ```
   stdin -> parse HookContext -> daemon POST /hook -> format output
   ```
4. Run TypeScript and Rust checks
