## What

Add a learning-pipeline contribution to `plugins/core/`. The core plugin owns platform primitives (knowledge, rule, lesson, agent, doc) and is responsible for the learning stage per RES-d6e8ab11 section 3. This contribution fills the `learning-pipeline` contribution point of the `agile-methodology` workflow skeleton.

The learning pipeline covers: lesson creation from implementation experience, retrospective analysis, pattern tracking across lessons, and recurrence detection to surface systemic issues.

## Knowledge Needed

- Read `plugins/core/orqa-plugin.json` for current manifest structure
- Read `plugins/core/workflows/` for existing workflow files (to understand format)
- Read existing contribution workflow files for format reference (e.g., `plugins/agile-planning/workflows/planning.contribution.workflow.yaml` from TASK-3)
- Read `plugins/agile-methodology/workflows/agile-methodology.workflow.yaml` (after TASK-2 rename) for the contribution point definition
- Read `plugins/core/schemas/` for lesson schema (the primary artifact of the learning pipeline)
- Read RES-d6e8ab11 section 3 for learning pipeline design

## Agent Role

Implementer — create contribution workflow file, update core manifest.

## Steps

1. Create `plugins/core/workflows/learning.contribution.workflow.yaml`:
   - `contributes_to.workflow`: `agile-methodology`
   - `contributes_to.point`: `learning-pipeline`
   - Define stages for the learning pipeline:
     - `capture`: lesson creation from implementation experience
     - `retrospective`: analysis of lessons for patterns
     - `pattern-tracking`: aggregation and trend detection across lessons
     - `recurrence-detection`: surfacing systemic issues that recur
2. Update `plugins/core/orqa-plugin.json`:
   - Add the learning contribution workflow to `provides.workflows`
   - Add `content` mapping for `orqa install` to copy the workflow to `.orqa/`
3. Run `npx tsc --noEmit` in `libs/cli`
