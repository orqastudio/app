## What

`agile-workflow` is the workflow-definition plugin. After this task it defines ONLY the agile-methodology skeleton with contribution point slots. Zero artifact type definitions.

Move ALL remaining artifact schemas and workflows OUT of agile-workflow to their correct homes:
- `decision`, `doc` schemas -> `plugins/core/` (platform primitives)
- `knowledge`, `rule`, `lesson`, `agent` schemas -> `plugins/core/` (if not already there)
- `vision`, `persona`, `pillar`, `pivot`, `idea` -> `plugins/agile-discovery/` (already done in TASK-5)
- Any remaining standalone workflows -> the plugin that owns the type definition

After completion, `plugins/agile-workflow/` contains ONLY:
- `agile-methodology.workflow.yaml` (the skeleton)
- `orqa-plugin.json` manifest (with empty or absent `provides.schemas`)
- Content: knowledge/rules/docs ABOUT the agile workflow model (not artifact type definitions)

## Knowledge Needed

- Read `plugins/agile-workflow/orqa-plugin.json` for current manifest to identify all remaining schemas
- Read `plugins/agile-workflow/schemas/` for remaining schema files
- Read `plugins/agile-workflow/workflows/` for remaining workflow files (excluding the skeleton)
- Read `plugins/core/orqa-plugin.json` to check what schemas core already defines
- Read `plugins/core/schemas/` to check for existing platform primitive schemas

## Agent Role

Implementer — move schema/workflow files, update manifests, verify nothing remains.

## Steps

1. Inventory all schema files in `plugins/agile-workflow/schemas/`
2. Inventory all workflow files in `plugins/agile-workflow/workflows/` that are NOT `agile-methodology.workflow.yaml`
3. For each remaining schema:
   - If it's a platform primitive (decision, doc, knowledge, rule, lesson, agent): move to `plugins/core/schemas/` and add to core manifest
   - If it's a discovery artifact (vision, persona, pillar, pivot, idea): verify already moved in TASK-5, delete if still present
   - If it belongs to another stage: move to the appropriate stage plugin
4. For each remaining standalone workflow (not the skeleton, not contributions):
   - Move to the plugin that now owns the type definition
5. Update `plugins/agile-workflow/orqa-plugin.json`:
   - Remove all entries from `provides.schemas` (or set to empty array)
   - Remove standalone workflow entries from `provides.workflows` (keep only the skeleton)
   - Keep content entries (knowledge, rules, docs about the agile model)
6. Update `plugins/core/orqa-plugin.json` to include any newly moved schemas and workflows
7. Verify `plugins/agile-workflow/` directory contains only: manifest, skeleton workflow, and content files
8. Run `npx tsc --noEmit` in `libs/cli`

- `ls plugins/agile-workflow/schemas/ 2>/dev/null | wc -l` should be 0 (directory empty or absent)
