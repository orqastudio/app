## What

`libs/cli/src/lib/graph.ts` is a complete TypeScript reimplementation of the Rust graph scanner. It walks `.orqa/`, parses frontmatter with the `yaml` library, builds `GraphNode[]`, infers types from path segments, and supports query/filter/stats. This duplicates `libs/validation/src/graph.rs` with divergent type inference logic (path-only vs frontmatter-first).

The connector (`connectors/claude-code/src/index.ts` line 13) re-exports these functions, extending the duplication surface.

## How

1. Create a daemon client utility in `libs/cli/src/lib/daemon-client.ts`
2. Implement daemon HTTP calls for `/query`, `/health` (stats), `/validate`
3. Migrate `orqa graph query` command to use daemon client
4. Migrate `orqa graph stats` command to use daemon client
5. Add fallback: if daemon is not running, shell out to `orqa-validation` binary
6. Update connector re-exports to point to new daemon-backed functions
7. Delete `libs/cli/src/lib/graph.ts` (or strip it to just the daemon client)

## Files

- `libs/cli/src/lib/graph.ts` — primary target for removal
- `libs/cli/src/commands/graph.ts` — CLI command that uses the scanner
- `connectors/claude-code/src/index.ts` — re-exports to update
- `libs/validation/src/daemon.rs` — daemon endpoints (already exist)
