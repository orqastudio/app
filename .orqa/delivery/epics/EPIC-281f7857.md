## Scope

From RES-55bacef1 sections 4 (Agent Architecture) and 8 (Token Efficiency):

- Agent spawn from generated prompts — replace CLAUDE.md loading with pipeline output
- Three-layer taxonomy: Universal Role + Stage Context + Domain Knowledge = Effective Agent
- Model tier selection — connector/integration defines available models, workflow coordinates mapping (not hardcoded to software domain)
- Role tool constraints per artifact type — Researcher gets web-search but only creates/edits research artifacts, Implementer gets shell but only edits code, etc.
- Token tracking hooks — per-request (input/output/cache), per-agent (total/utilization), per-task (tokens per deliverable), trends (7-day)
- Findings-to-disk summary format — orchestrator reads structured headers only (~200 tokens)
- Agent token budget enforcement — per-agent for prompt size, per-session for total cost
- Session metrics reporter — writes to `.state/token-metrics.jsonl`
