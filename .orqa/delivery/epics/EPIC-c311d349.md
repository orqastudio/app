## Context

Research (`tmp/team/graph-perf/research.md`) identified 4 performance bottlenecks in the three-layer graph query architecture (MCP Server -> Validation Daemon -> ArtifactGraph). The most critical issue was that `query_artifacts()` re-read every matching file from disk on every query, despite frontmatter already being cached in the in-memory `ArtifactGraph`.

All 4 fixes have been implemented and verified (571 Rust tests pass, zero warnings).

## Architecture

```
Claude Code (MCP client)
    |
    v  (JSON-RPC over stdio)
MCP Server (Rust: libs/mcp-server)
    |
    v  (HTTP REST to localhost:9258)
Validation Daemon (Rust: libs/validation, tiny_http)
    |
    v  (in-memory HashMap)
ArtifactGraph  <-- built by scanning .orqa/ files from disk
```

## Implementation (Completed)

### Fix 1: Cache body content in ArtifactNode
- Added `body: Option<String>` field to `ArtifactNode`, populated at graph-build time
- `artifact_from_graph_node()` reads cached `node.body` instead of disk
- Eliminates ~N disk reads per query (N = matching artifacts)
- Files: `libs/validation/src/graph.rs`, `libs/validation/src/parse.rs`

### Fix 2: O(1) HashMap lookup for ID queries
- `query_artifacts()` now tries `graph.nodes.get(id)` directly for exact-ID queries
- Falls through to prefix iteration only for org-mode qualified keys
- Changes `graph_resolve` from O(n) to O(1) for ~200 artifact graphs
- Files: `libs/validation/src/parse.rs`

### Fix 3: Daemon traceability endpoint
- Added `handle_traceability()` handler and `/traceability` route to daemon
- MCP `graph_traceability` tool now proxies through daemon instead of rebuilding graph from scratch
- Eliminates ~200 file reads per traceability call
- Files: `libs/validation/src/daemon.rs`, `libs/mcp-server/src/daemon.rs`, `libs/mcp-server/src/tools/graph.rs`

### Fix 4: Push search filter to daemon
- Added `search_filter: Option<&str>` to `query_artifacts()`
- Daemon `/query` endpoint accepts `search` parameter
- Removed client-side search filtering from MCP server
- Files: `libs/validation/src/parse.rs`, `libs/validation/src/daemon.rs`, `libs/mcp-server/src/tools/graph.rs`
