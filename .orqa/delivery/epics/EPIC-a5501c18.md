## Context

EPIC-9d781696 was marked done with ~60-70% of its scope unfinished. The remediation pass (2026-03-25) fixed the mechanical issues (daemon.rs, filenames, inverses, statusTransitions, workflows, CLAUDE.md, content_file paths). What remains is the connector rebuild and the verification/documentation/triage work that was never attempted.

The connector at `connectors/claude-code/` has 13 hook source files that call the Rust daemon via HTTP. The hooks work but are wired to old daemon endpoints. The new libs/cli/ modules (prompt-pipeline, knowledge-retrieval, agent-spawner, workflow-engine, gate-engine, token-tracker, budget-enforcer) are built but nothing in the connector uses them.

### Phase 2: Rewire Hooks to New Architecture

**TASK-3: Rewire prompt-injector to use prompt pipeline**

- Replace daemon `/content/behavioral` + `/content/agent` calls with prompt pipeline
- Import and call `generatePrompt()` from libs/cli/src/lib/prompt-pipeline.ts
- Keep connector-specific UX (session state checks, context line) in the hook
- Remove hardcoded CRITICAL_RULE_IDS, CATEGORY_RELEVANCE — the pipeline handles this
- The hook becomes: classify prompt → call pipeline → format systemMessage

**TASK-4: Rewire knowledge-injector to use knowledge retrieval**

- Replace manual agent-definition parsing with `retrieveKnowledge()` from libs/cli/src/lib/knowledge-retrieval.ts
- Keep semantic search layer (MCP IPC) as-is — it's a valid fallback
- Use the prompt registry for declared knowledge instead of hand-parsing employs relationships

**TASK-5: Replace rule-engine with workflow-guard evaluation**

- The rule-engine hook calls daemon `/hook` which evaluates rules and returns allow/block/warn
- Replace with workflow engine guard evaluation: load the artifact's workflow, evaluate guards for the current transition
- For Write/Edit: check if the artifact being modified has workflow constraints
- For Bash: either pass-through or apply lightweight safety checks
- The workflow engine already has all guard types (field_check, relationship_check, role_check)

**TASK-6: Remove/update remaining hooks**

- `findings-check.ts` (PostToolUse:TaskUpdate) — keep but update to use new findings format from agent-spawner

- `impact-check.ts` (PostToolUse:Write|Edit) — evaluate if still needed
- `artifact-enforcement.ts` (PreToolUse:Write|Edit) — evaluate overlap with rule-engine replacement
- `memory-redirect.ts` — evaluate if still needed
- `departure-detection.ts` — evaluate if still needed
- `save-context.ts` (PreCompact) — keep, useful
- `subagent-review.ts` (SubagentStop) — keep, useful
- `session-start.sh` — update for .state/ paths
- `stop-checklist.sh` — update for .state/ paths
- Update hooks.json to reflect final hook set
