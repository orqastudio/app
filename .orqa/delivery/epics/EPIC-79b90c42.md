# Centralised Logger Library

## Problem

The SDK logger (`libs/sdk/src/logger.ts`) has Svelte/Tauri peer dependencies, making it unimportable by the CLI, connector, and other Node consumers. The connector has its own telemetry.mjs that writes to a local file instead of the centralised logger. Multiple logging mechanisms exist across the codebase.

## Target State

- `libs/logger` — standalone package (`@orqastudio/logger`)
  - Rust crate: used by daemon, validation engine, LSP, MCP, search
  - TypeScript package: used by CLI, connector hooks, any Node process
  - Zero dependencies (works in Node + browser)
- SDK re-exports the TS logger for frontend use (bridge to Rust in-app)
- Debug tool consumes log output from all processes
- `tmp/hook-metrics.json` eliminated — all telemetry goes through the logger

## Repo

`https://github.com/orqastudio/orqastudio-lib-logger`

## Tasks

1. Create libs/logger submodule with Rust crate + TS package
2. Extract logger logic from libs/sdk/src/logger.ts
3. Add Rust logging crate (tracing-compatible)
4. Update SDK to import from @orqastudio/logger
5. Update CLI to use @orqastudio/logger
6. Update connector to use @orqastudio/logger (replace telemetry.mjs)
7. Wire into LSP, MCP, search, app backend
8. Delete old telemetry.mjs and tmp/hook-metrics.json writing
