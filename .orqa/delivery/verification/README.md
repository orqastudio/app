# Verification Records

Verification records (VER-NNN) are the final stage of the knowledge maturity pipeline. They prove that enforcement is working — that rules are being followed, standards are being met, and the system is healthy.

Each verification record tracks:
- **What** rule or standard is being verified
- **How** evidence is collected (automated checks, manual review, metrics)
- **When** verification runs (per-commit, per-session, per-epic, on-demand)

## Pipeline Role

Verification is the **sixth stage** of the knowledge maturity pipeline:

```
Observation → Understanding → Principle → Practice → Enforcement → Verification
```

A verification record closes the loop — it proves that enforcement is effective. If verification consistently fails, the pipeline has a gap somewhere upstream.
