## What's Missing

The SDK (`libs/sdk/`) is the primary API surface for plugins and the app UI. It exposes:

### Stores (12)
- `NavigationStore` ‚Äî routing, activity panel, artifact selection, hash-based navigation
- `ArtifactStore` ‚Äî nav tree, artifact loading, CRUD
- `SessionStore` ‚Äî conversation sessions, messages, streaming
- `ProjectStore` ‚Äî project settings, project path
- `ConversationStore` ‚Äî active conversation state
- `SettingsStore` ‚Äî user preferences, active sections
- `EnforcementStore` ‚Äî validation, integrity checks
- `ErrorsStore` ‚Äî error collection, display
- `LessonsStore` ‚Äî lesson pipeline
- `SetupStore` ‚Äî setup wizard state
- `ToastStore` ‚Äî notifications
- `PluginRegistry` ‚Äî plugin discovery, view registration

### Router
- `pushRoute` / `replaceRoute` ‚Äî hash-based routing
- `parseHash` / `currentRoute` ‚Äî route parsing
- `injectNavigation` ‚Äî SvelteKit integration (injection pattern)
- `navigateToArtifact` / `navigateToPluginView` / `navigateToActivity` etc.

### IPC
- Tauri invoke wrappers
- Type-safe command interfaces

### Plugin System
- View registration
- Shared module exposure (`window.__orqa`)
- Plugin lifecycle

### Utilities
- Frontmatter parsing
- Logger

## What's Needed

1. **SDK Reference Guide** ‚Äî each store's API, properties, methods, usage examples
2. **Plugin Developer Guide** ‚Äî how to use SDK from a plugin (navigation, data access, views)
3. **Knowledge artifacts** ‚Äî for agent context when building plugins or app UI
4. **Architecture doc** ‚Äî the injection pattern, store initialization lifecycle, data flow
