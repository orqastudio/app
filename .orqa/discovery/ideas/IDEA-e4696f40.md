## Motivation

OrqaStudio's frontend is Svelte + Tailwind, currently using shadcn-svelte for base components. As the UI grows (artifact browser, conversation panel, devtools, onboarding), we need to build many domain-specific components efficiently without sacrificing quality.

21st.dev offers an interesting pattern: a curated component registry combined with AI-powered generation (Magic Chat) that produces production-ready React components. It sits on top of shadcn/ui rather than replacing it, and has an MCP server for integration with AI coding tools.

The framework (React) doesn't match ours (Svelte), but the *pattern* is worth studying:

1. **Curated registry** — community-contributed, quality-gated components that AI tools can reference
2. **AI generation from registry** — generating new components that follow the same conventions as the registry
3. **MCP integration** — making the component library accessible to AI assistants during development
4. **shadcn as foundation** — building domain-specific components on top of a solid base library

## Questions to Answer

- Does shadcn-svelte have the same extensibility patterns that make 21st.dev possible on React?
- Could we build an OrqaStudio component registry that serves both human developers and AI assistants?
- Would an MCP server for our component library accelerate development with Claude Code?
- Is the right move to contribute to/extend shadcn-svelte, or to build our own registry layer?
- What's the quality bar for AI-generated Svelte components vs hand-written ones?

## Initial Assessment

21st.dev itself is not directly usable (React-only). But the architecture — registry + AI generation + MCP — aligns with OrqaStudio's principles: plugin-composed (P1), generated not loaded (P3), and the connector pattern for tool integration. Worth a research spike to evaluate the pattern, not the specific product.
