## Motivation

During the monorepo migration and plugin framework work, enforcement mechanisms (pre-commit hooks, linter configs, the prompt injector) have been in flux. Code written during this period may not comply with the documented standards. A full audit catches drift before it becomes the new normal.

## Scope

- Rust: clippy pedantic, no unwrap/expect/panic in production, function size limits, error handling
- TypeScript: strict mode, no `any`, no `@ts-ignore`, Svelte 5 runes only
- Both: no TODO comments, no commented-out code, no hardcoded fake data
- Component purity: no `invoke()` in `$lib/components/`
- Test coverage: 80%+ per module

## Approach

Run `make check` and treat warnings as failures. Then a manual review for patterns that linters don't catch (component purity, store patterns, function sizes).

**Fix, don't just report.** Any drift found during the audit gets fixed in the same epic — not deferred. The audit produces a clean codebase, not a findings report.
