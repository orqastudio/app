## Problem

Claude Code's permission model currently presents a binary choice in practice:

1. **Default mode** — agents are frequently interrupted for permission to edit files, slowing down delegated work and breaking flow
2. **dangerously-skip-permissions** — agents edit anything without approval, risking unintended changes to critical files (governance artifacts, config, lock files)

Neither extreme is ideal for an agentic workflow where agents are delegated implementation tasks with clear scope.

## What to Investigate

### Claude Code Permission Mechanisms

- `allowedTools` in settings.json — which tools agents can use without prompting
- Path-based edit restrictions — can specific directories be protected while others are open?
- Tool-level granularity — e.g., allow `Edit` on `ui/**` but require approval for `.orqa/**`
- `mode` parameter on Agent spawning — `acceptEdits`, `bypassPermissions`, `default`, `dontAsk`, `plan`, `auto`
- Per-agent permission profiles — can different agent roles have different permission levels?

### Desired Outcome

A permission configuration where:

- **Implementation agents** can freely edit source code (`ui/`, `backend/`, `sidecar/`, `libs/`) without prompting
- **Governance artifacts** (`.orqa/process/rules/`, `.orqa/process/agents/`, `schema.json`) require approval
- **Config files** (`package.json`, `Cargo.toml`, `project.json`, `settings.json`) require approval
- **Git operations** (force push, reset, branch delete) always require approval
- **Documentation** (`.orqa/documentation/`) is freely editable by Writer agents

### Questions

1. Does Claude Code support path-based permission rules, or only tool-level?
2. Can the `mode` parameter on Agent spawning achieve per-role permissions?
3. What's the interaction between settings.json `allowedTools` and agent `mode`?
4. Are there hooks (PreToolUse) that could implement path-based gating?
5. How do other teams using Claude Code in agentic workflows handle this?

## Research Needed

- Claude Code documentation on permission model
- Current settings.json allowedTools configuration in this project
- PreToolUse hook capabilities for path-based gating
- Real-world patterns from other agentic Claude Code setups
