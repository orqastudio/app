## Context

Currently, artifact views follow a standard master-detail pattern. The roadmap view (from software-kanban) is the only custom view. Plugins should be able to provide alternate ways to view and interact with artifacts — for example, a timeline view for epics, a dependency graph for tasks, or a canvas view for wireframes.

## Proposed Approach

Plugins declare alternate views in their `orqa-plugin.json` manifest:

- Each view targets one or more artifact types
- Views are Svelte components distributed with the plugin
- The app registers them and makes them selectable via a view switcher on the artifact panel
- This connects to the navigation override system (IDEA-fd2a66bd) where users could set a preferred default view per artifact type

This extends the existing pattern where software-kanban provides a roadmap view, making it a first-class plugin capability rather than a one-off.
