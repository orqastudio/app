## Decision

Rename `.state/` to `.state/` across the project. Session state, token metrics, and hook telemetry are operational state data scoped to a session — not disposable temporary files.

## Rationale

`.state/` semantically means "safe to delete at any time." But `session-state.md` bridges context between sessions, `token-metrics.jsonl` tracks efficiency over time, and `hook-metrics.json` provides audit data. Naming the directory `tmp` invites accidental deletion and miscommunicates the purpose of its contents.

`.state/` communicates: "ephemeral per-session, but not disposable while a session is active."

## Consequences

- All references to `.state/` in rules, hooks, scripts, and code must be updated
- `.gitignore` entry changes from `.state/` to `.state/`
- Session state path becomes `.state/session-state.md`
- Token metrics path becomes `.state/token-metrics.jsonl`
- Team findings path becomes `.state/team/<name>/`
- Hook scripts that read/write `.state/` must be updated
