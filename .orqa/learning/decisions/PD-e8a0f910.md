## Decision

The app's native Rust enforcement engine is the primary implementation. The CLI companion plugin is a secondary port. All enforcement features are designed for the app first.

## Rationale

- The app has full control over the enforcement pipeline (process gates, skill injection, tool approval)
- The CLI plugin operates within the constraints of the CLI tool's hook system
- Designing for CLI first limits what the app can do; designing for app first gives the richest experience
- In the app, enforcement sits above the agent layer — it doesn't matter which agent is running
- The plugin exists to bring the rules architecture to CLI users as best we can

## Consequences

- Enforcement features are implemented in Rust first, then ported to plugin hooks/scripts
- The app must never depend on or reference the CLI plugin
- Plugin capabilities are a subset of app capabilities, not the other way around
