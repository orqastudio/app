# AD-059: Central LSP/MCP Registration via Plugin Manifests

## Decision

Plugins declare their LSP and MCP servers in `orqa-plugin.json` using `lspServers` and `mcpServers` fields. The Claude Code connector generates its `.lsp.json` and `.mcp.json` from the aggregated plugin manifests at session start. The app can read the same declarations for its own agent context.

## Format

```json
{
  "name": "@orqastudio/plugin-svelte",
  "provides": {
    "lspServers": {
      "svelte": {
        "command": "svelteserver",
        "args": ["--stdio"],
        "extensionToLanguage": { ".svelte": "svelte" }
      }
    },
    "mcpServers": {
      "svelte-tools": {
        "command": "node",
        "args": ["${PLUGIN_ROOT}/mcp-server.js"]
      }
    }
  }
}
```

## How It Works

1. Each plugin declares servers in its `orqa-plugin.json`
2. The connector's session-start hook scans all installed plugins
3. Aggregates all `lspServers` and `mcpServers` declarations
4. Generates `.lsp.json` and `.mcp.json` in the connector directory
5. Claude Code reads the generated files (standard plugin discovery)
6. Future: the app's Rust plugin loader reads the same declarations

## Why

- Single source of truth — servers declared once in the plugin manifest
- Central registry — both app and Claude Code consume the same declarations
- Plugin composability — installing a language plugin automatically provides its LSP/MCP
- No manual `.lsp.json`/`.mcp.json` editing
