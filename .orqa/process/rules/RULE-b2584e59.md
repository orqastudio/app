When any artifact is created or any insight is captured, the orchestrator MUST immediately trace it to all affected artifacts before moving on. Observing a connection without acting on it is a rule violation.

## The Rule

After creating or updating ANY artifact (lesson, rule, decision, knowledge, idea), ask:

> **Where does this get used?**

Then update every affected artifact:
- **Milestones** — does this artifact inform a gate question or readiness criterion?
- **Pillars** — does this insight suggest a new or updated gate question?
- **Rules** — does this lesson warrant promotion or update to an existing rule?
- **Epics** — does this change scope, priority, or acceptance criteria?
- **Other lessons** — does this lesson connect to or supersede an existing lesson?

This tracing is not optional. It is part of completing the task. A task that produces a new artifact is NOT done until affected artifacts have been updated or the orchestrator has explicitly confirmed no affected artifacts exist.

## What This Means for the Orchestrator

**"Worth noting"** is a violation. If something is worth noting, note it — update the artifact. Flagging something verbally in a response and not recording it is the same as not recording it.

**"Eligible for promotion"** is a violation. If a lesson is eligible for promotion, promote it. Observing eligibility and not acting is the same as not observing it.

**"I'll do that next"** without doing it is a violation. If tracing is deferred and then not completed, the gap is the same as if the tracing never happened.

The correct pattern is: artifact created → "where does this get used?" → affected artifacts updated → task complete.

## Enforcement

**Prompt injection:** The connector's prompt injector (`connectors/claude-code/src/hooks/prompt-injector.ts`) injects behavioral rules into agent context. The trace-to-usage reminder should be included as a behavioral rule message on rules that enforce artifact tracing, so it is injected automatically when governance-related prompts are classified.

> For every new artifact or insight captured this turn: ask "where does this get used?" and update all affected artifacts (milestones, pillars, rules, epics, lessons) before considering the task complete.

**Session start protocol:** The orchestrator's session start should include a check: for any lesson in the previous session state marked `recurrence >= 2` or `maturity: correction`, verify it has been traced to its affected artifacts before beginning new work.

## FORBIDDEN

- Saying "worth noting" in a response without immediately recording the note in an artifact
- Saying "eligible for promotion" without promoting in the same turn
- Completing a task that produced a new artifact without asking "where does this get used?"
- Deferring impact tracing to a future task or session without a tracked task in `.orqa/delivery/tasks/`
