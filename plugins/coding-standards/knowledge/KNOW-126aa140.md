# Quality Check Runner

## Running Checks

```bash
orqa check              # Run all configured checks
orqa check eslint       # Run a specific tool
orqa check clippy       # Run a specific tool
```

## How It Works

1. Discovers installed plugins with `tools` in their `orqa-plugin.json`
2. Runs each tool's command with the plugin-specified config file
3. Returns pass/fail for each with captured output
4. Exits non-zero if any check fails — blocks the commit

## Plugin Tool Format

Plugins declare tools in their manifest:

```json
{
  "provides": {
    "tools": {
      "eslint": {
        "command": "npx eslint",
        "configFile": ".eslintrc.json",
        "configFormat": "json"
      }
    }
  }
}
```

Each tool entry specifies the shell command to run, the config file path relative to the project root, and the config format (json, yaml, toml). The runner resolves config paths and passes them to the tool.

## Available Check Tools

Standard plugins provide these tools:

| Plugin | Tool | What It Checks |
|--------|------|----------------|
| typescript | eslint | Linting and style rules |
| typescript | svelte-check | Svelte component type-checking |
| rust | clippy | Rust lint warnings |
| rust | cargo-fmt | Rust formatting |
| coding-standards | prettier | Code formatting (non-Rust) |

## Integration Points

- **Pre-commit hook**: `orqa check` runs automatically before every commit. Failing checks block the commit.
- **CI pipeline**: Same command runs in CI for consistency.
- **Agent workflow**: Implementer agents must run `orqa check` before reporting task completion.

## Rules

- Always run `orqa check` before committing — never use `--no-verify` to skip
- Never run raw `cargo clippy`, `npx eslint`, etc. directly — always go through `orqa check`
- Fix all failures before proceeding — do not ignore warnings promoted to errors
