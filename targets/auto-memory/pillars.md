---
name: Three pillars
description: The three product pillars with gate questions — equal weight, conflicts flagged to user
type: project
---

**Pillar 1: Clarity Through Structure** — Making thinking, standards, and decisions visible and structured.
Gate: Does this make governance visible? Does it produce structured knowledge? Does it enforce understanding before action?

**Pillar 2: Learning Through Reflection** — The system and its users improve over time through structured retrospection.
Gate: Does this capture lessons? Does it track improvement? Does it feed retrospectives back into governance?

**Pillar 3: Purpose Through Continuity** — The system maintains coherence between intention and action across time.
Gate: Does this prevent drift? Does it reduce cognitive burden? Is scope drift visible and explicit?

**Why:** Pillars are equal. When they conflict, the conflict is flagged to the user — agents do not prioritise one over another.

**How to apply:** Every feature, task, and design decision should serve at least one pillar. If it serves none, question whether it belongs.
